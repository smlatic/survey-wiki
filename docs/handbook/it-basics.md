# Chapter 11: IT Basics

## Computers, Networks, and Connectivity

You don't need to be an IT specialist, but a solid grasp of basic computer and network concepts will save you a lot of headaches. Understanding the difference between LAN (Local Area Network) and WAN (Wide Area Network), knowing how to troubleshoot a flaky Wi-Fi connection, or handling a minor software crash -- these skills keep you productive when problems come up.

As a trainee, you'll start with the basics:

- **Rebooting Hardware:** Sometimes turning the system off and on really does fix the issue.
- **Checking Cable Connections:** A loose Ethernet or USB cable can look like a serious failure -- until you plug it back in properly.
- **Verifying IP Addresses:** If you can't connect to a device or server, confirm you're on the correct subnet and IP range.

Decisions about vessel network architecture or advanced cybersecurity fall to IT specialists and management. But your willingness to learn and handle smaller problems makes you a more valuable team member.

## Managing Software & Updates

Not glamorous, but staying on top of software updates is key to smooth operations. Survey applications evolve rapidly, introducing new features or fixing bugs that could affect data quality or workflow. Senior staff typically schedule major upgrades and test new versions before rolling them out. You can still help by:

- **Reporting Glitches Promptly:** If you notice odd behavior -- a button that doesn't work, a sudden crash -- tell someone. Early reporting can prevent others from losing work.
- **Tracking Version Numbers:** Keep a record of which versions of key software are installed. It's easy to forget whether the echo sounder software got updated last month or last year.
- **Flagging Minor Issues:** Even small hiccups (like a misaligned GUI element) can be clues to bigger compatibility problems. A quick screenshot and note helps senior staff determine if it's a one-off or a sign of something deeper.

Over time, you'll learn which updates are routine and which need careful coordination (like major OS changes). Staying aware of this makes you a useful resource for version-related questions.

## Cybersecurity Basics

Phishing attacks, malware, ransomware -- these threats are real and can compromise sensitive project data fast. You're not expected to deploy firewalls or develop security policy, but you play a role by:

- **Avoiding Suspicious Emails:** If an email looks off -- weird grammar, unexpected attachments, alarming links -- consult your supervisor or IT.
- **Never Sharing Passwords:** It's tempting to give a colleague your login "just this once," but that undermines security protocols.
- **Reporting Anything Odd:** A strange pop-up, a suspicious external drive -- speak up. Small anomalies can be early signs of larger security risks.

Your company's reputation depends on safeguarding data, and a single misstep can cost millions or damage client relationships. Even basic habits like logging out of your workstation when you step away keep the entire project safer.

## Troubleshooting Common Tech Issues

Technology hiccups are inevitable. A calm approach and a few basic troubleshooting steps make all the difference:

1. **Restart Software:** Closing and reopening the application often clears temporary glitches.
2. **Check Cable Connections:** Jiggling a loose connector or swapping a questionable cable can restore functionality in seconds.
3. **Verify Licenses and Resources:** Some survey applications require valid licenses or might run slowly if your CPU or RAM is maxed out. A quick look at Task Manager (Windows) or Activity Monitor (macOS) can reveal if a program is hogging resources.
4. **Close Unneeded Tasks:** If your PC starts lagging during a survey, check running processes. Shutting down non-essential apps frees up memory and CPU.
5. **Document What You Tried:** Write down the steps you took, even if they didn't work. This helps the next person pick up where you left off without duplicating effort.

If these steps don't resolve the issue, or if the problem is more serious (network outage, corrupted data), call in the experts. Stay calm, follow standard procedures, and learn from the situation. Over time, you'll build a troubleshooting instinct that serves you whether you're onshore or at sea.
